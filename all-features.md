## Детали фич

### F-03 — Авторизация координатора и администратора
Координатор и администратор входят в систему по логину/паролю (или SSO). Сессии, refresh-токены, защита от перебора.

### F-04 — Персонализированная подборка
UI выбора эмбрионов врачом; генерация уникальной ссылки; хранение связи пациент ↔ эмбрионы; отправка ссылки пациенту (email/мессенджер); срок действия подборки.

**Критерии готовности**: пациент видит только свои эмбрионы; ссылка без токена → редирект; врач видит статус — открыта ли подборка.

### F-05 — Карточка эмбриона
Данные доноров (возраст, рост, образование, этничность); предсказанный фенотип ребёнка; генетический скрининг и риск-факторы; качество эмбриона и стадия развития; фото эмбриона (опционально); дисклеймер: не является мед. заключением.

**Критерии готовности**: скрытые поля не попадают в API-ответ пациенту; карточка рендерится без фото (null-state); дисклеймер присутствует всегда; событие `view_embryo` логируется.

### F-06 — Каталог и фильтрация
Список карточек из подборки пациента; фильтры: цвет глаз, рост, этничность, генетика; сортировка по приоритету / алфавиту; пустое состояние каталога; URL-синхронизация фильтров.

**Критерии готовности**: фильтры применяются без перезагрузки; применение фильтра логируется; при 0 результатах — понятный empty state; фильтры сбрасываются отдельной кнопкой.

### F-07 — Избранное и сравнение
Добавление / удаление из избранного; хранение избранного (сессия или сервер); режим сравнения: 2–3 эмбриона; сравнительная таблица по ключевым полям; подсветка различий.

**Критерии готовности**: избранное сохраняется при обновлении страницы; нельзя добавить >3 в сравнение; событие `add_to_favorites` логируется; сравнение работает с 2 и с 3 эмбрионами.

### F-08 — Заявка на эмбрион
Форма: имя, контакт, выбранный эмбрион; явное согласие на обработку ПДн (152-ФЗ); интеграция с CRM (webhook / API); подтверждение отправки пользователю; защита от двойной отправки.

**Критерии готовности**: без галочки согласия форма не отправляется; после отправки эмбрион помечается `reserved`; событие `submit_application` логируется; ошибка CRM не теряет данные заявки.

### F-09 — Compliance UI
Cookie-баннер с выбором политики; страница политики обработки ПДн; дисклеймер на всех медицинских экранах; согласие перед отправкой любой формы; язык интерфейса — русский (ФЗ о гос. языке).

**Критерии готовности**: баннер появляется при первом входе; без принятия политики аналитика не запускается; дисклеймер не скрывается прокруткой.

### F-10 — Панель врача
Список пациентов и их подборок; создание и редактирование подборки; просмотр заявок от пациентов; доступ к скрытым полям (пол, расш. генетика); ответы в чате; аналитика по своим подборкам.

**Критерии готовности**: врач видит только своих пациентов; скрытые поля недоступны на стороне пациента; заявка меняет статус эмбриона.

### F-13 — Панель администратора
CRUD эмбрионов и доноров; ручная смена статуса эмбриона; управление пользователями и ролями; логи действий администратора; системные метрики и health-check.

**Критерии готовности**: удаление эмбриона с активной заявкой — запрещено; все действия администратора логируются; изменение статуса требует подтверждения.
